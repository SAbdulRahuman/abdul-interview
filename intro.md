## Expanded Version (~3–4 min)

Hi, I'm **Abdul Rahuman S**, a software engineer with over **15 years of experience** in developing cloud native microservices on Kubernetes / Openshift clusters. Experienced designing high-performance distributed systems. My career spans from low-level systems programming all the way up to leading cloud-native platform engineering at scale.

### Early Career — Firmware & Systems (Wipro / IBM, 2010–2014)

I started my career at **Wipro Technologies**, working on-site for **IBM** as a firmware engineer. I was part of a 10–15 member team developing firmware for IBM's **RAID SAS Switch Module** — working with PowerPC WCS, writing code in **Assembly, C, and C++**, and building kernel modules for SCSI, Fibre Channel, and RAID subsystems. One of my key contributions was implementing **3TB hard disk support** for IBM RSSM using the SCSI-10 protocol — I owned the full cycle from design through development, testing, and client delivery. This role gave me a strong foundation in **systems programming, storage internals, and performance optimization** at the hardware level.

### Mid Career — Systems Software Engineering (HPE, 2014–2020)

I then moved to **Hewlett Packard Enterprise**, where I spent about six years in systems software engineering. I was part of a 20-member team building a **Data Center Simulation (DCS)** platform that simulated servers, interconnects, and image streamers for HPE Synergy and OneView products. I redesigned the DCS architecture by **replacing in-memory Redis with ArangoDB**, which significantly reduced the memory footprint and enabled containerization. I also optimized I/O performance to achieve **30K+ ops/sec** through multi-threading, thread pooling, and queue-based processing. Additionally, I built a **Remote DCS Tool** using Bash, iproute2, and iptables that enabled multi-environment HA testing without requiring physical hardware — saving significant lab costs.

### Current Role — Lead Cloud Native Engineer (HPE, 2020–Present)

For the last **five-plus years at HPE**, I've been working as a **Lead Cloud Native Engineer**, focused on Kubernetes and OpenShift platforms for large-scale **telecom workloads (4G/5G)**. Here are some key highlights:

- **Performance at Scale:** Scaled the 4G IHSS platform to handle **75K TPS** through container scheduling optimization, pod auto-scaling, and Linux performance tuning on OpenShift.
- **Kubernetes Operators:** Designed and deployed **production-grade Kubernetes Operators** that automate the full lifecycle management of 4G/5G telecom services in OpenShift clusters.
- **Microservices:** Built high-throughput microservices — including ENS, SNMP Adapter, and IHSS Operator — processing **real-time event streams via Kafka** and RESTful APIs. The ENS microservice was adopted by a major client in a **$2–3M deal**.
- **Observability:** Led the 5G Advanced Observability project — designing end-to-end **Prometheus/Grafana monitoring, KPI metrics, logging, and tracing pipelines** using OpenTelemetry.
- **Security:** Implemented end-to-end **TLS encryption** using OpenSSL — securing external APIs, inter-service communication, and PostgreSQL connections, while hardening the Linux OS security posture.
- **CI/CD & Automation:** Automated deployment of microservices and Operators using **Helm charts, Ansible, and Jenkins** pipelines following GitOps practices.
- **Technical Leadership:** Served as a **Subject Matter Expert** in a 14-member team — collaborated with architects on design discussions, developed POCs/MVPs for client and chief architect review, and led technical enablement after approval.
